## 1. Phase 목표

지망 작가가 정리되지 않은 스토리 아이디어에서 출발하여, 시각적 단일 트랙 타임라인·캐릭터 맵·장면 단위 AI 생성을 활용해 다중 장면 AI 보조 초고를 완성할 수 있음을 증명한다 — 그리고 그 초고가 버리지 않고 편집할 만한 수준인지를 검증한다.

## 2. 범위 설정 근거

### 왜 이 범위인가?

전체 제품 비전에는 멀티트랙 타임라인, 에피소드 구성 레이어, 월드 맵, 리비전 도구, 내보내기, 고급 AI 기능(변형 생성, 톤 슬라이더, AI 서프라이즈)이 포함된다. Phase 1은 이 모든 것을 걷어내고 한 가지 질문에 답한다: **핵심 루프가 작동하는가?**

핵심 루프: 아이디어 입력 → 시각적 구조화 → 스토리 배치 → 장면 산문 생성 → 편집 및 다듬기. 이 루프가 사용자가 계속 편집하고 싶은 초고를 만들어내지 못한다면, 나머지는 의미가 없다. 멀티트랙 타임라인, 에피소드 구분, 리비전 도구는 먼저 유효성이 증명된 핵심 루프의 배율기(force multiplier)일 뿐이다.

### 왜 이 하위 범위인가?

1. **입력 + 자동 구조화(REQ-001, REQ-002, REQ-004)는 협상 불가.** 제품의 첫 순간 약속은 "아이디어를 가져오세요, 우리가 구조화합니다"이다. 이것 없이는 사용자는 기존과 같은 빈 페이지 문제에 직면한다.

2. **멀티트랙 타임라인(REQ-009, REQ-010, REQ-011, REQ-012, REQ-013, REQ-014, REQ-015)은 시각적 구조.** 타겟 장르(회귀물, 로맨스판타지, 무협)에서 동시 전개 — 주인공이 훈련하는 동안 적이 음모를 꾸미는 — 는 일상적인 패턴이다. 병렬 트랙, 병합/분기 포인트, 동시 이벤트의 수직 정렬을 포함한 멀티트랙 타임라인을 Phase 1에 포함한다. 이를 빼면 사용자가 실제로 쓰는 스토리에 비해 시각적 구조가 불완전하게 느껴지기 때문이다.

3. **캐릭터 관계 맵(REQ-024, REQ-025, REQ-026)은 컨텍스트 조립의 핵심.** AI 초고 품질은 캐릭터 인식에 의존한다. 생성 컨텍스트에 캐릭터 데이터가 없으면 출력은 범용적이 된다. 캐릭터 맵은 또한 온보딩 과정에서 "보세요, AI가 제 이야기를 이해하고 있어요"라는 실감나는 시각적 순간을 제공한다.

4. **AI 초고 생성(REQ-034, REQ-035)은 수익 엔진.** 사용자가 비용을 지불할 기능이다. 지불 의향과 초고 품질을 검증하기 위해 MVP에 반드시 포함되어야 한다.

5. **기본 에디터(REQ-042, REQ-043)는 루프를 닫는다.** 사용자는 AI 출력을 읽고 편집하고 개선할 수 있어야 한다. 방향 기반 편집 요청("긴장감 더", "대화 추가")은 비작가 타겟 사용자의 스킬 갭을 직접 해결하는, 가장 적은 노력으로 산문을 개선하는 방법이므로 포함된다.

6. **설정 바(REQ-007, REQ-008)는 글로벌 컨텍스트를 제공.** 장르, 톤, 시대, 시점 설정은 AI 출력 품질에 극적인 영향을 미친다. 설정 없이는 모든 장면 생성 시 수동 프롬프트 컨텍스트가 필요해져 "당신의 구조가 곧 프롬프트" 가치 제안을 무력화한다.

### 이번 릴리스에서 무엇을 학습할 수 있는가?

- 자동 구조화 결과물이 사용자가 출발점으로 삼기에 충분히 좋은가? (아니면 버리고 수동으로 다시 구성하는가?)
- 사용자가 시각적 타임라인과 상호작용하는가, 아니면 무시하고 바로 에디터로 가는가?
- AI 생성 장면 초고가 편집할 만한 수준인가? (사용자 편집 후 생성 텍스트 유지율)
- 실제 사용 패턴에서 사용자당 AI 비용은 얼마인가?
- 사용자가 여러 장면을 완성하는가, 아니면 하나 생성하고 떠나는가?

| P0 | REQ-004 | 시스템이 모든 입력을 초기 설정·타임라인·캐릭터 맵으로 자동 구조화한다 | 사용자 입력으로부터 생성: (a) 설정 바 값(장르, 테마, 시대/배경, 시점, 톤); (b) 핵심 플롯 포인트를 나타내는 순서화된 장면이 있는 타임라인 — 해당되는 경우 동시 이벤트를 병렬 트랙에 배치; (c) 캐릭터 노드와 추론된 관계가 있는 캐릭터 맵. 모든 생성 요소는 편집 가능. 시스템이 추론한 내용의 간략 요약을 보여주고 사용자에게 검토·조정을 안내. | 핵심 "매직 모먼트." 이것 없이는 제품은 또 하나의 빈 캔버스에 불과. |

**REQ-004 인수 기준:**
- 식별 가능한 캐릭터와 플롯 포인트가 있는 200자 이상 입력이 주어지면, 시스템이 30초 이내에 최소 5개 타임라인 장면(병렬 스토리라인이 감지되면 하나 이상의 트랙에 걸쳐), 2명의 캐릭터(최소 1개 관계 포함), 채워진 설정 바를 생성한다.
- 모호한 입력(예: "사랑 이야기")이 주어지면, 구조 생성 전 2-3개 명확화 질문을 한다.
- 모든 자동 생성 요소(장면, 캐릭터, 설정 값)는 생성 후 개별 편집이 가능하다.
- 첫 결과가 맞지 않을 경우 동일 입력에서 전체 구조를 재생성할 수 있다.

- 장르를 "로맨스"에서 "스릴러"로 변경한 후, 다음 AI 생성 장면 초고가 사용자가 다른 컨텍스트에서 변경을 언급하지 않아도 스릴러 관습(긴장감, 페이싱, 어휘)을 반영한다.
- 설정 변경 전에 초고가 생성된 장면에 "생성 이후 설정 변경됨"이라는 미묘한 표시가 나타난다.

**REQ-010 인수 기준:**
- 기존 두 장면 사이에 단일 인터랙션("+" 클릭)으로 새 장면을 추가할 수 있다.
- 3번 위치의 장면을 7번 위치로 드래그할 수 있다; 4-7번 장면이 왼쪽으로 이동하고 이동된 장면이 7번에 위치한다. 시각적으로 500ms 이내에 완료.
- 한 트랙에서 다른 트랙으로 장면을 드래그할 수 있다; 장면의 트랙 할당이 업데이트되고 수직 정렬이 조정된다.
- 기존 콘텐츠가 있는 장면을 삭제하면 확인 대화상자가 표시된다. 빈 장면 삭제 시 확인 불필요.

**REQ-011 인수 기준:**
- 단일 인터랙션으로 새 트랙을 생성할 수 있다. 트랙 레이블은 선택적이며 편집 가능.
- 장면이 없는 트랙만 삭제할 수 있다(또는 포함된 모든 장면의 삭제를 확인).
- 타임라인이 최대 5개 병렬 트랙을 레이아웃 저하 없이 표시한다.

**REQ-012 인수 기준:**
- 한 장면에서 드래그하여 다른 트랙에 새 장면을 만들어 분기 포인트를 생성할 수 있다.
- 여러 트랙의 장면을 하나의 하류 장면에 연결하여 병합 포인트를 생성할 수 있다.
- 분기/병합 포인트가 일반 순차 연결과 시각적으로 구분된다.

**REQ-013 인수 기준:**
- 서로 다른 트랙에서 같은 내러티브 시간 위치의 장면이 1그리드 단위 이내로 수직 정렬된다.
- 사용자가 트랙 할당을 해제하지 않고 장면을 가로로 드래그하여 수동으로 정렬을 조정할 수 있다.

**REQ-014 인수 기준:**
- 장면 상세 패널을 열어도 타임라인에서 이동하지 않는다 — 패널이 타임라인 위 또는 옆에 나타난다.
- 패널에 나열된 캐릭터는 캐릭터 맵에서 가져온다. 캐릭터 맵에 없는 캐릭터를 패널에서 추가하면 새 캐릭터 항목 생성을 안내한다.
- 플롯 요약은 글자 수 제한 없는 다중 행 텍스트 입력을 지원한다.

- 캐릭터 카드는 자유 입력 필드당 최소 500자를 지원한다.

**REQ-026 인수 기준:**

- 관계 레이블과 유형(실선/점선/화살표)은 생성 후 편집 가능하다.
- AI 생성 초고에서 사용자가 장면의 플롯 요약에 관계를 명시적으로 언급하지 않아도 관계 컨텍스트가 반영된다(예: 라이벌 간 긴장감, 동맹 간 따뜻함).

### 3.5 AI 초고 생성

| 우선순위 | REQ-ID | 요구사항 | Phase 1 상세 | 근거 |
|----------|--------|----------|--------------|------|
| P0 | REQ-034 | 사용자가 장면을 선택하고 "AI 초고 생성"을 실행할 수 있다 | 사용자가 타임라인에서 장면을 선택하거나 에디터에서 열고 "생성" 버튼을 누른다. 시스템이 단일 산문 초고를 생성(Phase 1은 하나의 변형만 생성; 복수 변형은 Phase 2로 연기). 출력 길이 목표 장면당 1,500-3,000 한글 글자. 생성 중 예상 시간 표시와 로딩 상태. 생성 실패 시 재시도 옵션이 있는 명확한 오류 메시지 표시. | 핵심 가치 전달 순간. 시각적 구조를 읽을 수 있는 산문으로 변환하는 지점. |
| P0 | REQ-035 | 시스템이 구조화된 컨텍스트에서 AI 프롬프트를 자동 조립한다 | Phase 1 컨텍스트 조립 포함: (a) 글로벌 설정(장르, 톤, 시대, 시점); (b) 현재 장면의 플롯 요약, 캐릭터, 장소, 분위기 태그; (c) 장면에 관련된 캐릭터의 캐릭터 카드 및 관계 데이터; (d) 선행 장면의 AI 압축 요약(전체 텍스트가 아닌 — 각 장면의 초고 완료 시 생성·캐싱된 요약); (e) 다음 장면의 제목과 요약(있는 경우) — 내러티브 연속성을 위해. 연기된 컨텍스트 소스: 복선 링크, 에피소드 위치, 에피소드 종료 훅 유형. 참고: 다른 트랙의 동시 이벤트는 Phase 1 컨텍스트 조립에 포함(REQ-013을 통해). | "당신의 구조가 곧 프롬프트." 자동 조립이 제품의 기술적 핵심 — 사용자가 모르게 시각적 편집을 프롬프트 엔지니어링으로 변환한다. |

**REQ-034 인수 기준:**
- 플롯 요약과 최소 1명의 지정 캐릭터가 있는 장면에서 "생성"을 누르면 30초 이내에 산문 초고가 생성된다.
- 플롯 요약이 없는 장면에서 "생성"을 누르면 장면 제목, 설정, 주변 컨텍스트를 기반으로 초고를 생성하지만, 더 나은 결과를 위해 플롯 요약 추가를 제안한다.
- 생성된 산문은 설정된 장르와 톤에 적합한 자연스러운 한국어이다.
- 사용자가 단일 액션과 확인 대화상자로 재생성(현재 초고 덮어쓰기)할 수 있다.

**REQ-035 인수 기준:**
- 캐릭터 카드에 "냉철하고 계산적"으로 묘사된 캐릭터 A와 "라이벌" 관계인 캐릭터 B가 등장하는 장면에서, 성격과 관계 역학 모두를 반영한 산문이 생성된다.
- 타임라인 10번째 위치의 장면이 이전 장면의 관련 사건(압축 요약을 통해)을 모순 없이 참조한다.
- 선행 장면의 초고를 변경하고 후속 장면의 초고를 재생성하면, 업데이트된 이전 콘텐츠가 요약 컨텍스트에 반영된다.

- 사용자가 방향 기반 편집을 실행취소하여 이전 버전을 복원할 수 있다.

## 5. Phase 1 사용자 여정

### 여정 1: 아이디어에서 첫 AI 초고까지 (주요 여정)

핵심 가설을 검증하는 여정이다. 이 여정을 완료한 사용자는 Phase 1의 전체 가치 루프를 경험한 것이다.

1. 사용자가 Narrex를 열고 새 프로젝트를 생성한다.
2. 사용자가 진입점을 선택한다:
   - **자유 텍스트:** 스토리 아이디어를 타이핑하거나 붙여넣는다 — "실패한 기사가 어린 시절로 회귀하는 회귀물"부터 캐릭터 노트와 플롯 아이디어가 담긴 여러 페이지 분량까지 무엇이든.
   - **파일 가져오기:** .md나 .txt 파일(또는 Notion .zip 내보내기)을 프로젝트 생성 영역에 드래그한다.
3. 시스템이 입력을 처리한다:
   - 식별 가능한 캐릭터와 플롯 포인트가 있는 경우 → 시스템이 설정 바(장르, 톤 등), 장면이 있는 단일 트랙 타임라인, 관계선이 있는 캐릭터 맵을 생성. 간략 요약 표시: "X명의 캐릭터, Y개의 플롯 포인트를 찾았습니다. 여기 스토리 구조입니다."
   - 입력이 너무 모호한 경우(예: "사랑 이야기") → 시스템이 2-3개 명확화 질문(예: "장르는?", "주인공은?", "핵심 갈등은?")을 한 후 구조 생성.
4. 사용자가 작업 공간을 본다: 상단/중앙에 장면이 있는 가로 멀티트랙 타임라인(병렬 스토리라인은 별도 트랙에), 사이드 패널의 캐릭터 그래프, 상단의 설정 바(접이식). 모든 것이 자동 구조화에서 미리 채워져 있다.
5. 사용자가 자동 생성된 구조를 검토한다:
   - 의도와 맞지 않는 장면의 이름을 변경.
   - 캐릭터 노드를 클릭하고 성격 설명을 더 구체적으로 편집.
   - 시스템이 놓친 장면을 위해 기존 두 장면 사이에 새 장면 추가.
   - 맞지 않는 장면 삭제.
   - 장면을 드래그하여 순서 재배치.
   - 두 캐릭터 간 관계선 추가("라이벌").
6. 사용자가 관심 있는 장면을 선택한다(예: "주인공이 회귀 능력을 발견").
7. 장면 상세 패널이 열린다. 사용자가 자동 생성된 플롯 요약을 검토하고 편집: "주인공이 전투에서 죽은 후 12살 몸으로 눈을 뜬다. 어린 시절 방에 있다. 불신, 그러다 서서히 이것이 현실임을 깨닫는다. 주먹을 꽉 쥐며 끝 — 이번에는 다를 것이다."
8. 사용자가 "AI 초고 생성"을 누른다.
9. 시스템이 로딩 인디케이터를 표시. 30초 이내에 에디터 패널에 산문 초고가 채워진다(1,500-3,000 한글 글자).
   - 생성 실패 시 → 시스템이 재시도 옵션과 함께 오류 표시.
10. 사용자가 초고를 읽는다. 회귀 순간을 잘 포착했지만 감정 톤이 너무 과장되었다. 사용자가 두 번째 단락을 선택하고 "더 절제된, 내면 독백, 덜 극적으로"라는 방향을 입력하고 "적용"을 누른다.
11. 시스템이 선택된 구간만 재생성한다. 새 버전은 더 차분하고, 더 내면적이다.
12. 사용자가 약간의 수동 편집을 한다 — 단어 선택을 바꾸고, 생각 한 줄을 추가.
13. 장면 상태가 "비어있음"에서 "편집됨"으로 업데이트. 사용자가 타임라인에서 장면이 채워지는 것을 본다.
14. 사용자가 다음 장면을 선택하고 7-13단계를 반복한다.

**이탈 위험 및 완화:**
- **3단계 (자동 구조화 품질):** 생성된 구조가 사용자의 비전을 빗나가면 포기할 수 있다. 완화: 모든 요소가 편집 가능; 시스템이 검토를 안내; 재생성 가능.
- **9단계 (첫 초고 품질):** AI 초고가 통째로 버릴 정도로 나쁘면 핵심 가치 제안이 실패. 완화: 프롬프트에 풍부한 컨텍스트 포함(설정, 캐릭터 데이터, 플롯 요약); 방향 기반 편집으로 재작성 없이 조정 가능.
- **10단계 (편집 스킬 갭):** 사용자가 초고를 어떻게 개선해야 할지 모를 수 있다. 완화: 방향 기반 편집(REQ-043)으로 산문 기술 대신 의도 설명("긴장감 더")으로 대체.

**Phase 1과 전체 비전의 차이:** 전체 제품에서 8단계는 톤 슬라이더와 함께 2-3개 변형을 생성한다. Phase 1에서는 하나의 초고를 생성한다. 출력이 마음에 안 드는 사용자는 재생성하거나 방향 기반 편집을 사용할 수 있다. 멀티트랙 타임라인은 Phase 1에 포함되어 사용자가 처음부터 병렬 스토리라인을 구성할 수 있다.

### 여정 2: 캐릭터 맵 구축

1. 사용자가 작업 공간에서 캐릭터 맵 패널을 연다.
2. 초기 구조화에서 자동 생성된 캐릭터 노드와 기본 관계선이 보인다.
3. 주인공 노드를 클릭. 자동 추론된 필드(이름, 성격, 외모 — 일부는 빈약하거나 추론된 것으로 표시)가 있는 캐릭터 카드가 열린다.
4. 사용자가 캐릭터 카드를 풍부하게 만든다: 상세한 성격 추가("규율적이지만 전생의 죄책감에 시달림"), 외모 추가("장신, 마른 체형, 검술 훈련 사고로 왼손에 흉터"), 비밀 추가("미래 기억으로 숨겨진 보물의 위치를 안다"), 동기 추가("다가올 재앙에서 가족을 보호하는 것").
5. 사용자가 원본 입력에 없는 새 캐릭터를 생성: 나중에 적대자가 될 주인공의 소꿉친구.
6. 주인공에서 새 캐릭터로 관계선을 드래그. "소꿉친구"라고 레이블링. 선 유형을 실선(긍정)으로 설정.
7. 사용자가 타임라인으로 돌아가 두 캐릭터가 등장하는 장면을 선택하고 AI 초고를 생성. 초고가 자연스럽게 우정 역학과 주인공의 숨겨진 지식을 반영.

**Phase 1과 전체 비전의 차이:** 시간적 관계 추적 없음. 관계는 단일 정적 상태. 사용자가 특정 타임라인 시점에서 "소꿉친구"가 "앙숙"이 되는 것을 표시할 수 없다. AI는 모든 장면에 현재 관계 상태를 사용.

### 여정 3: 프로젝트 이어서 작업

1. 사용자가 다음 날 Narrex로 돌아온다.
2. 기존 프로젝트를 연다. 타임라인이 진행 상황을 보여준다: 3개 장면이 "편집됨"(채워짐), 2개가 "AI 초고"(부분 채워짐), 8개가 "비어있음."
3. 사용자가 마지막으로 편집한 장면 다음의 빈 장면을 선택하며 이어서 작업한다.
4. 장면 상세 패널에 플롯 요약을 쓰고, 캐릭터를 지정하고, AI 초고를 생성한다.
5. 초고가 이전 장면의 사건(완료된 장면의 압축 요약을 통해)을 참조하며 내러티브 연속성을 유지한다.
6. 사용자가 초고를 편집하고 장면 상태가 업데이트된다.

**이탈 위험:** 사용자가 하던 작업을 잊었다. 완화: 장면 시각적 상태로 한눈에 진행 상황을 파악. 타임라인이 지속적인 스토리 맵 역할.

### Phase 1 종료 기준

Phase 1이 검증되었고 Phase 2로 진행해야 하는 조건:

1. **핵심 루프 완료율 30% 초과.** 프로젝트를 시작한 신규 사용자의 최소 30%가 첫 AI 생성·편집 장면에 도달. 30% 미만이면 기능 추가 전에 해결해야 할 근본적인 온보딩 또는 가치 전달 문제가 있다.

2. **텍스트 유지율 40% 초과.** 사용자가 편집 후 AI 생성 텍스트의 최소 40%를 유지. 40% 미만이면 초고가 수정되지 않고 버려지는 것 — "처음부터 쓰지 말고 수정하라" 가설이 성립하지 않는다.

3. **다중 장면 참여가 존재.** 핵심 루프를 완료한 사용자의 최소 15%가 5개+ 장면을 생성·편집. 이는 도구가 신기함에 그치지 않고 지속적인 가치를 사용자가 인식함을 시사.

4. **사용자당 AI 비용이 지속 가능.** 활성 사용자당 월 평균 AI 비용이 $5 미만(Basic 플랜 $12/월 가격에서 ~40% 비용 상한에 부합).

5. **치명적 품질 차단 요인 없음.** AI 생성 한국어 산문이 테스트 사용자의 최소 70%에게 정성 피드백(구조화된 설문 또는 인터뷰)에서 "수정할 만한 수준"으로 평가.

처음 세 기준 중 하나라도 8주 후에 달성되지 않으면, Phase 2로 진행하지 않고 Phase 1을 진단·반복해야 한다.

## 7. Phase 1 가정 & 리스크

### 가정

| 가정 | 검증 계획 | 틀렸을 때의 영향 |
|------|-----------|-----------------|
| 장면 단위 한국어 산문 생성이 버리지 않고 편집할 만한 수준이다. | 출시 전: 3개 장르(회귀, 로맨스판타지, 무협)에 걸쳐 50+ 테스트 장면을 생성하고 5-10명의 타겟 사용자에게 평가 의뢰. 출시 후: 텍스트 유지율 추적. | 치명적. 틀리면 전체 제품 가설 실패. 완화: 복수 모델 제공자 테스트, 프롬프트 엔지니어링 투자, 파인튜닝 고려. |
| 사용자가 시각적 타임라인에 혼란 없이 참여할 것이다. | 출시 후: 사용자가 타임라인과 상호작용(장면 추가/이동/삭제)하는지 아니면 무시하고 에디터만 사용하는지 추적. | 높음. 타임라인이 무시되면 차별화 요소 상실. 완화: 타임라인 UI 간소화, 인라인 안내 제공, "리스트 뷰" 폴백 고려. |
| 자유 텍스트에서의 자동 구조화가 유용한 출발점을 생성한다. | 출시 후: 자동 생성된 요소(장면, 캐릭터)가 사용자 편집 후 얼마나 유지되는지 vs. 삭제 후 재생성되는지 추적. | 높음. 사용자가 버리고 처음부터 재구성하면 자동 구조화가 거짓 약속. 완화: 구조화 프롬프트 개선, "처음부터 구조화" 경로를 명시적으로 추가. |
| 간단한 장면별 요약이 10+ 장면에 걸친 AI 생성 연속성을 위한 충분한 컨텍스트이다. | 개발 중: 요약 기반 컨텍스트로 1, 5, 10, 15번째 장면의 생성 품질 테스트. 모순과 놓친 플롯 스레드 확인. | 중간. 요약에서 중요한 세부사항이 누락되면 후반 장면 품질 문제. 완화: 최근 장면을 전문 텍스트로 더 많이 포함, 핵심 플롯 포인트에 보존 태그. |
| 지망 작가가 AI 글쓰기 도구를 시도할 것이다(인지도와 의향). | 출시 전: 타겟 오디언스와 메시징·포지셔닝 검증. 출시 후: 가입-프로젝트 생성 전환율 추적. | 중간. 타겟 사용자가 AI 글쓰기 도구에 회의적이면 유입이 병목. 완화: "AI 작가"가 아닌 "AI 보조가 있는 스토리 구조화 도구"로 포지셔닝. |

### 리스크

| 리스크 | 심각도 | 가능성 | 완화 |
|--------|--------|--------|------|
| 타겟 장르에서 한국어 AI 산문 품질이 "편집할 만한" 기준 이하 | 치명적 | 중간 | 출시 전 복수 프로바이더(OpenAI, Anthropic, Google) 대상 타겟 사용자와 품질 벤치마크. 프로바이더 비종속 게이트웨이로 빠른 A/B 테스트 가능. 장르별 프롬프트 엔지니어링. 출시 시점에 품질 미달이면 출시를 지연하고 프롬프팅 반복. |
| 멀티트랙 타임라인이 첫 사용자에게 온보딩 복잡성을 추가 | 중간 | 낮음 | 프로젝트 생성 시 합리적 기본값(단일 트랙) 제공; 자동 구조화가 동시 이벤트를 감지하거나 사용자가 명시적으로 추가할 때만 병렬 트랙 등장. 첫 조우 시 인라인 힌트로 트랙 개념 설명. |
| 컨텍스트 윈도우 제한으로 15+ 장면 프로젝트에서 품질 저하 | 높음 | 중간 | Phase 1 프로젝트는 10-20 장면 예상. 처음부터 요약 기반 컨텍스트 압축 구현. 장면 위치별 생성 품질 모니터링. 저하 감지 시 Phase 2 전 압축 품질에 투자. |
| 사용자가 장면 하나 생성하고 떠남 — 도구가 워크플로우가 아닌 신기한 것 | 높음 | 중간 | UX에서 다중 장면 참여를 유도: 첫 생성 후 "다음 장면 생성하시겠어요?" 제안. 타임라인에 진행 상황 표시. 생성→두 번째 생성 전환을 조기 경고 메트릭으로 추적. |
| 재생성과 방향 기반 편집으로 인해 사용자당 AI 비용이 모델링보다 높음 | 중간 | 낮음 | 첫날부터 사용자당 토큰 사용량 모니터링. 내부 비용 알림 설정. 예측 초과 시 정확한 사용 데이터 수집을 위해 베타 사용자에게도 생성 제한 고려. |
| 파일 가져오기(Notion 내보내기) 파싱이 엣지 케이스에서 실패하여 첫인상 악화 | 낮음 | 중간 | 잘 테스트된 제한된 파일 형식(.md, .txt, Notion .zip) 지원. 미지원 형식에 명확한 오류 메시지. 자동 구조화가 부분 파싱 결과를 우아하게 처리. |

## 8. Phase 1 타임라인 & 마일스톤

| 마일스톤 | 설명 |
|----------|------|
| M1: 기반 | 프로젝트 데이터 모델, 인증, 기본 작업 공간 레이아웃. 자동 구조화 파이프라인(입력 → 설정 + 타임라인 + 캐릭터 맵). |
| M2: 타임라인 + 캐릭터 맵 | 장면 CRUD와 드래그앤드롭이 있는 단일 트랙 타임라인. 노드 그래프, 캐릭터 카드, 관계선이 있는 캐릭터 맵. 장면 상세 패널. |
| M3: AI 생성 + 에디터 | 컨텍스트 조립이 있는 AI 초고 생성. 방향 기반 편집 요청이 있는 기본 텍스트 에디터. 장면 상태 관리. |
| M4: 통합 + 다듬기 | 엔드투엔드 플로우 테스트. 분석 계측(REQ-051). 성능 최적화. 버그 수정. |
| M5: 베타 출시 | 20-50명 타겟 사용자 초대 출시. 정성 피드백 수집. 성공 메트릭 모니터링. |
| M6: 반복 | 핵심 피드백 대응. 자동 구조화 품질, 생성 품질, UX 마찰 포인트 반복 개선. |
